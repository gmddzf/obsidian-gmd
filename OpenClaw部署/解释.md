### `npm install` 做了什么

当你在项目目录执行 `npm install` 时：

1. **找清单**：npm 会在当前目录查找 `package.json` 文件
2. **读依赖**：解析 `dependencies` 和 `devDependencies` 字段，列出所有要安装的包
3. **下载包**：连接 npm 官方仓库，下载对应版本的包到本地 `node_modules` 文件夹
4. **生成锁文件**：创建 / 更新 `package-lock.json`（或 `pnpm-lock.yaml`），记录精确版本，保证下次安装环境一致

#### 3. 为什么必须 “进入项目目录”

- `npm install` 只会在**当前工作目录**找 `package.json`
- 如果你不在 `openclaw` 目录里，npm 找不到这个清单文件，就不知道要装什么
- 只有进入项目根目录，npm 才能识别这是一个 Node.js 项目并执行依赖安装
